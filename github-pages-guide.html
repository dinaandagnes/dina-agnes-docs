Overview
How It Works in Plain English

Your GitHub repository is like a folder in the cloud. Any HTML file you upload becomes a live web page automatically. Every file follows the same URL pattern:

dinaandagnes.github.io/dina-agnes-docs/your-filename.html

Upload a file, get a permanent URL. No configuration, no build steps, no waiting beyond about 60 seconds.

Task 01
Adding a New HTML Document

Do this whenever Claude creates a new HTML document you want to host permanently.

1
Go to your repository
Open your browser and go to:
github.com/dinaandagnes/dina-agnes-docs
2
Click "Add file" → "Upload files"
The green "Add file" button is near the top right of the file list. Click it and select "Upload files" from the dropdown.
3
Drag your HTML file into the upload box
Drag directly from your Downloads folder. You can upload multiple files at once. Do NOT drag a folder — drag individual files only.
Important
File names become part of the URL. Use lowercase letters and hyphens instead of spaces. Example: june-content-plan.html not "June Content Plan (Final).html"
4
Click "Commit changes"
Scroll down and click the green "Commit changes" button. Leave the commit message as default.
5
Wait ~60 seconds then open your URL
dinaandagnes.github.io/dina-agnes-docs/your-filename.html
Done
Copy this URL and paste it into Notion, share with Agnes, or add it anywhere. Permanent and never expires.
Task 02
Updating an Existing Document

When Claude creates an updated version of a document you already have hosted. The URL stays exactly the same.

1
Go to your repository
Go to github.com/dinaandagnes/dina-agnes-docs
2
Click on the file you want to update
Find the file in the list and click its name.
3
Click the pencil icon → Select all → Delete
Pencil ✏️ icon is top right of the file content. Click it, then Cmd+A and delete everything.
Easier alternative
Delete the old file and upload the new one with the exact same filename. URL stays identical.
4
Paste the new HTML content
Open the new HTML file in TextEdit (right-click → Open With → TextEdit), select all, copy, paste into GitHub editor.
5
Click "Commit changes"
The page at the same URL updates within 60 seconds.
Task 03
Deleting a Document
1
Click on the file in your repository
Find it in the file list and click its name.
2
Click ⋯ → Delete file
The ⋯ icon is top right of the file view, next to the pencil icon.
3
Commit the deletion
Click "Commit changes". The file and its URL are permanently gone.
Note
Remove any Notion links pointing to that URL — they'll show a 404 error once deleted.

FAQ
Common Questions
How long until a new file is live?
Usually 30–90 seconds. If it's not loading after 2 minutes, do a hard refresh (Cmd+Shift+R on Mac).
Will these URLs ever break or expire?
No. As long as your GitHub account and repository exist, the URLs are permanent. GitHub Pages has been free for years with no sign of that changing.
Can Agnes access these documents?
Yes — just share the URL. Anyone with the link can view the page. No login required to view.
Can I add other file types — PDFs, images?
Yes. Upload them the same way. PDFs and images get their own URL too. Example: dinaandagnes.github.io/dina-agnes-docs/my-document.pdf
How many files can I have?
Free GitHub plan allows up to 1GB per repository. You'd need hundreds of thousands of HTML files to hit that. Effectively unlimited for your use case.
Can I use a custom domain like docs.dinaandagnes.com?
Yes, and it's free. Go to repository Settings → Pages → Custom domain. You'll also need a DNS record in your domain registrar. Ask Claude to walk you through it when ready — it takes about 10 minutes.
