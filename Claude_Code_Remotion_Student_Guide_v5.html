Claude Code + Remotion Student Setup Guide

Everything you need to install once, then use every single day to create motion graphics with AI. Follow each step in order — do not skip ahead.

macOS & Windows Claude Code Desktop Remotion Studio Node.js LTS
💡
What are we setting up?

You are going to connect two tools together: Remotion and Claude Code.

Remotion is the engine that creates your motion graphics. It runs on your computer and generates video animations using code. You do not need to write the code yourself — that is Claude's job.

Claude Code is the AI. You describe what you want in plain English, Claude writes the code, and Remotion turns that code into a video you can export and drop straight into CapCut.

Think of it like this: Remotion is the studio, Claude Code is the animator you hired.

📋
Before you start — what you need

You need four things installed before anything else works. If you already have one, skip it. If you are not sure, the steps below will check for you.

💻

A Mac or Windows computer

This entire workflow runs on your desktop, not in a browser.

🟢

Node.js LTS — the engine that runs Remotion

Download the LTS version (not "Current") from nodejs.org. Run the installer, click through all the defaults.

🔧

Git — a background tool Claude Code needs

Download from git-scm.com. You will never open Git directly — it just needs to exist on your machine so Claude Code can use it.

🤖

Claude Code — the desktop app

Download from claude.ai/download. This is the AI app you will use every day. Free to install, requires a Claude account.

🔩
One-time setup
Do this once only

You only do these steps once ever. After today, skip straight to the Daily Workflow section every time you want to create.

01

Verify Node.js installed correctly

Confirm everything is ready before creating your project.

Open your command line app — Terminal on Mac, or Command Prompt on Windows (search "cmd" in the Start menu to find it). Type the command below and press Enter. You should see a version number printed back, like v20.11.0. Any number is fine as long as something appears.

# Check Node.js is installed node -v

Then check npm (it comes with Node.js automatically):

npm -v
💡 If you see "command not found" or "not recognised" after either of these, go back and re-install Node.js from nodejs.org, then restart your Terminal / Command Prompt and try again.
02

Verify Git installed correctly

Make sure Git is available on your machine.

Still in Terminal, type this and press Enter. You should see a version number like git version 2.39.0.

git --version
💡 If you see "command not found" or "not recognised", go back and install Git from git-scm.com, restart your Terminal / Command Prompt, and try again.
03

Create your Remotion project

Build the folder where all your motion graphics projects will live.

This command creates a new folder on your computer where all your motion graphics projects will live. Type it exactly as shown and press Enter.

npx create-video@latest

It will download some files first, then ask you a series of questions. Answer them exactly like this:

QuestionYour answer
TemplateChoose Blank
Project nameType ai-motion-lab and press Enter
Install Tailwind?Yes
Install Skills?Yes
ScopeChoose Project
Symlink?Yes (or press Enter to accept default)
04

Move into your new project folder

Navigate into the folder and install all project dependencies.

Now navigate your Terminal / Command Prompt into the folder that was just created:

cd ai-motion-lab

Then install all the project dependencies (this may take a minute or two):

npm install

You will see a lot of text scroll past. That is normal. Wait for it to finish and return you to a blank prompt.

05

Teach Claude Code how Remotion works

This one command stops Claude from making avoidable mistakes.

Run this command now, while you are still inside the ai-motion-lab folder:

npx remotion skills add

This command downloads a set of instructions that tell Claude Code exactly how Remotion works — things like how to structure animations, what properties exist, and how everything fits together. Without it, Claude is working from general knowledge and will make more mistakes, write broken code more often, and need more corrections from you. Running this once means Claude starts with the right information from the beginning.

💡 You only need to run this once per project. It is quick and runs quietly in the background. Wait for it to finish before moving to the next step.
06

Set up Git inside your project

Claude Code needs Git to be active before it will work.

Claude Code needs Git to be active inside your project folder before it will work. You only do this once. Run these three commands one at a time, pressing Enter after each:

# Step 1 — initialise Git git init
# Step 2 — mark all current files git add .
# Step 3 — save the first snapshot git commit -m "initial setup"
💡 This is required. If you skip it, Claude Code will show a "Git is required" error when you try to use it. These commands are harmless and only take a few seconds.
07

Launch Remotion Studio

Start the live preview window where you will watch your motion graphics build in real time.

Make sure you are still inside the ai-motion-lab folder in your Terminal / Command Prompt, then run:

npm run dev

After a few seconds, your browser should open automatically and show the Remotion Studio interface. If it does not open automatically, go to http://localhost:3000 in your browser.

⚠️ Do not close this Terminal / Command Prompt window or the browser tab. Remotion Studio must stay running the entire time you are working. If you accidentally close it, just run npm run dev again.
08

Open Claude Code and point it at your project

Connect the AI to your project folder so it can see and edit your files.

Open the Claude Code desktop app (the one you downloaded earlier — it looks like the Claude logo).

When it opens, it will ask you to select a folder. Navigate to and select your ai-motion-lab folder. Claude Code will ask you to allow access — click Allow.

💡 Not sure where the folder is? On Mac it is usually in your home folder (e.g. /Users/YourName/ai-motion-lab). On Windows it is usually in your user folder (e.g. C:\Users\YourName\ai-motion-lab). It was created wherever your Terminal / Command Prompt was pointing when you ran the setup command.

You are now fully set up. You should have Remotion Studio open in your browser and Claude Code open as a desktop app, both pointing at the same ai-motion-lab folder.

🎬
Daily workflow — how to create
Every session

From today onwards, this is the only routine you need. Setup is done. Now you just create.

1

Open your command line and navigate to your project

Every session starts here.

Open Terminal (Mac) or Command Prompt (Windows) and run:

cd ai-motion-lab
2

Start Remotion Studio

Get the live preview running before you start working.

npm run dev

Wait for your browser to open Remotion Studio, then leave that window running. You need to open a second command line window for the next steps.

💡 How to open a second window: On Mac, press Cmd+T in Terminal to open a new tab, or Cmd+N for a new window. On Windows, search "cmd" in the Start menu to open a fresh Command Prompt, or press Ctrl+Shift+T if you are using Windows Terminal.
⚠️ Do not use the Remotion window for anything else. Keep it dedicated to running Remotion.
3

Open Claude Code and select your project folder

Make sure Claude is pointing at the right place.

Open the Claude Code desktop app. If it does not automatically open the ai-motion-lab folder, use the folder picker to select it again.

4

Describe what you want to Claude in plain English

No code required — just explain your idea clearly.

Type your animation idea into Claude Code exactly as you would explain it to a person. You do not need to know any code. Examples:

Example
"Create a rolling number animation from 0 to 100,000 over 5 seconds on a transparent background. The final number should be in our brand green #02E482."
Example
"Create a subscriber count graphic with glassmorphism cards, soft shadows, a black background, and green accents. Animate the numbers from zero."

The more specific you are, the better the result. Mention colours, timing, sizes, and style if you know them.

5

Watch the preview update live

See your animation take shape in real time without touching any code.

Claude Code writes the code. The Remotion Studio browser tab updates in real time as it works. You can see your animation take shape without doing anything else.

If you want to tweak something — a colour, the timing, the position — just type the change in plain English back into Claude Code. You do not need to touch any code files manually.

6

Export your finished animation

Render your file ready to drop straight into CapCut.

When you are happy with the preview, go to Remotion Studio in your browser and click the Render button. Choose your output format (MP4 or transparent WebM for overlays), choose your export location, and click Render.

Your file is ready to import directly into CapCut.

Troubleshooting

Nothing happened after I ran npx create-video@latest

It may be downloading quietly in the background. Wait 30 seconds — it sometimes takes a moment before the setup questions appear.

If still nothing after a minute, check your internet connection and try again. If it says "permission denied", try running Terminal as administrator (Windows) or add sudo before the command (Mac).

Claude Code shows a "Git is required" error

This means you skipped or did not complete Step 6. Navigate to your project folder in Terminal / Command Prompt and run these three commands in order:

git init then git add . then git commit -m "initial setup"

After the third command, try Claude Code again.

Remotion Studio is not opening in my browser

After running npm run dev, go to your browser manually and type http://localhost:3000 in the address bar. That should open it directly.

If you see an error saying port 3000 is already in use, another app is blocking it. Either close that app or run npm run dev -- --port 3001 and visit http://localhost:3001 instead.

The preview is not updating when Claude Code writes code

Make sure both Claude Code and Remotion Studio are pointing at the same ai-motion-lab folder. If they are in different folders, Claude's changes will not appear in the preview.

Also check that the npm run dev Terminal / Command Prompt window is still running and not showing an error. If it stopped, run it again.

I accidentally closed the Remotion Terminal window

No problem. Open a new Terminal (Mac) or Command Prompt (Windows), navigate back to your project with cd ai-motion-lab, and run npm run dev again. Your project files are safe — nothing is lost.
